# 更新日志

本文件记录 **官网仓库**（heart-box-site）相对重要的界面与工程变更。  
站内「下载页」里的示例更新列表仍由 `lib/data/changelog.ts` 维护，与本文档无关。

格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，版本号可与 `package.json` 对齐；无发版计划时可用日期作为小节标题。

---

## [未发布]

### 变更

- **布局与移动端**：全站 section 纵向间距整体收紧约 10%～20%；标题区与卡片栅格 `gap`、内边距分级优化。
- **导航**：`md` 以下改为顶栏 + 侧滑菜单（奶油色抽屉、安全区、ESC/遮罩关闭、打开时锁定滚动）。
- **首页**：Hero 小屏单栏排版（标题 → 文案 → 按钮 → 纸卡预览），首屏信息更集中；功能卡、场景区、获取预览区同步收紧。
- **HeartBox 互动区**：小屏输入区与匣子纵向排列，「放进去」全宽易点按；Modal 适配小屏安全区与最大高度。
- **Notes**：顶区高度与字阶优化；筛选条小屏横向滚动；心事卡与列表间距收紧。
- **Product / Download**：理念卡、能力区、时间轴、FAQ、平台卡、版本说明与更新日志区块的间距与字号按断点分级；能力区小屏统一为「文案在上、预览在下」。
- **全局**：`viewportFit: cover`；`globals.css` 增加 `safe-x` / `safe-t` / `safe-b`；`Container` 小屏水平内边距略减；页脚留白收紧。

### 新增（较早一批合入本提交的改动摘要）

- 视觉精修：首页 Hero 纸感预览、功能卡统一图标与阴影、HeartBox 飞入/成功提示/匣子呼吸与 Modal 收纳感、Notes 纸条气质与空状态、Product FAQ 折叠与键盘操作、Modal 滚动锁定与 ESC 关闭等。
- 无障碍与动效：`focus-visible`、部分组件 `prefers-reduced-motion`、全站纸张线/轻噪点工具类。

## 以后怎么更新这份日志

1. **有用户可见的改动**（新页面、大改版、重要修复）时，在 `[未发布]` 下按类型追加一条（`新增` / `变更` / `修复` / `移除`）。
2. **准备发版或打标签**时：
   - 把 `[未发布]` 整块改名为版本号 + 日期，例如：`## [0.2.0] — 2026-03-27`；
   - 在顶部重新放一个空的 `## [未发布]`；
   - 可选：同步 bump `package.json` 的 `version`。
3. **与 App 版本无关**：官网仓库可独立迭代；若需和 App 版本对齐，在条目中写清「对应 App x.y.z」即可。

提交时建议单独带上 changelog，便于历史检索：

```bash
git add CHANGELOG.md README.md
git commit -m "docs: 更新 CHANGELOG"
```
